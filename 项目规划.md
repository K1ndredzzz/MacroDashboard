## GCP资源利用策略

**核心思路：用GCP做数据密集型任务，轻量服务部署到你的2核4G服务器**

- **BigQuery**（重点消耗额度）：存储历史宏观数据，做复杂查询和事件冲击模拟计算
- **Cloud Functions**：定时抓取数据源（FRED API、World Bank、IMF、各国央行API）
- **Cloud Storage**：存储原始数据备份和模拟结果
- **Pub/Sub**：数据更新事件流
- **Cloud Scheduler**：触发定时任务

## 技术架构

```
GCP侧：
├── Cloud Scheduler → Cloud Functions（数据采集）
├── BigQuery（数据仓库 + 分析引擎）
├── Cloud Storage（数据湖）
└── Pub/Sub（实时数据流）

你的服务器侧（Docker Compose）：
├── n8n（工作流编排 + 监控）
├── Web前端（React/Vue + ECharts/D3.js）
├── API Gateway（FastAPI/Express）
├── Redis（缓存层）
└── PostgreSQL（元数据 + 用户配置）
```

## 功能模块

**仪表盘核心指标**
- 利率曲线（美债、欧债、中债）
- 汇率矩阵（主要货币对）
- 大宗商品（原油、黄金、铜）
- 通胀指标（CPI、PPI、核心通胀）
- 就业数据（失业率、非农）
- GDP增长率（主要经济体）
- PMI指数（制造业/服务业）

**事件冲击实验室**
- 情景模拟：利率冲击、汇率贬值、油价暴涨
- 历史回测：选择历史事件（如2008金融危机、疫情冲击）查看各指标响应
- 相关性分析：动态计算指标间的滚动相关系数
- VAR模型：简单的向量自回归预测

## 数据源建议

**免费API**
- FRED（美联储经济数据）
- World Bank Open Data
- IMF Data
- ECB Statistical Data Warehouse
- Yahoo Finance（市场数据）
- Alpha Vantage（有限额度）

**抓取策略**
- 高频数据（汇率、商品价格）：每小时
- 中频数据（利率、股指）：每日
- 低频数据（GDP、CPI）：每周检查更新

## n8n工作流设计

1. **健康监控流**：检查GCP Functions状态、数据新鲜度、API配额
2. **数据同步流**：从GCP BigQuery拉取聚合数据到本地PostgreSQL
3. **告警流**：数据异常、服务宕机时发送通知
4. **备份流**：定期备份配置和用户数据

## Docker部署结构

```yaml
services:
  - n8n（端口5678）
  - frontend（Nginx + 静态文件）
  - api（后端服务）
  - redis（缓存）
  - postgres（本地数据库）
```

## 成本优化点

- BigQuery前10GB查询/月免费，设计好查询避免重复扫描
- Cloud Functions前200万次调用免费
- Cloud Storage前5GB免费
- 用Materialized Views预计算常用指标
- 本地Redis缓存减少GCP查询

## 开发优先级

1. 先搭建数据采集管道（GCP Functions + BigQuery）
2. 建立基础仪表盘（3-5个核心指标）
3. 部署到你的服务器，验证架构
4. 逐步添加更多指标和事件实验室功能
5. 最后优化n8n自动化运维
